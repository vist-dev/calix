# コア概念

## 差分（Diff）

Calix における状態管理の最小単位。ある時点の状態からの変更を表現する。

### 構造

差分は 2 つの要素で構成される。

- **set** — 追加または更新されたキーと値のペア
- **remove** — 削除されたキーの一覧

### 例

エフェクトの「強度を 0.5 → 0.8 に変更し、フェードアウトを削除した」場合の差分:

```
set:    { "intensity": "0.8" }
remove: [ "fade_out" ]
```

### 差分の生成と適用

- **生成**: 変更前の状態と変更後の状態を比較して差分を算出する。直接差分を手で組み立てる操作は想定していない
- **適用**: 既存の状態に差分を適用すると、新しい状態が得られる。remove → set の順で処理される

値は全て `String` 型として保存される。数値や真偽値などの型情報は呼び出し元が管理する。Calix にとっては全てが文字列である。

---

## コミット（Commit）

差分に時間的な文脈を与えたもの。一つのコミットは「ある時点で、あるサブモジュールに対して、この差分を適用した」という事実を記録する。

### コミットが持つ情報

- **ID** — UUIDv4 による一意識別子。既存 ID の再利用は禁止されている
- **親コミット ID** — 直前のコミットへのポインタ。最初のコミットは親を持たない
- **第二親コミット ID** — マージコミットの場合のみ値を持つ。マージ元ブランチの先端コミットを指す
- **差分** — このコミットが表す変更内容
- **サブモジュール ID** — このコミットが属する編集要素
- **グローバル順序** — タイムスタンプとシーケンス番号の組み合わせ
- **メッセージ** — 人間が読むための変更説明

### コミットの不変性

一度作成されたコミットは変更されない。リベースは元のコミットを書き換えるのではなく、新しいコミットを生成する。

### コミットチェーンと状態の再構築

コミットはルートから先端に向かって差分を積み上げることで状態を再構築する。

```
[ルート] ──→ [C1] ──→ [C2] ──→ [C3 (HEAD)]
  {}      {x:1}   {x:1,y:2}  {x:9,y:2}
```

C3 の状態を知りたい場合:
1. ルートまで遡ってチェーンを構築する（C1 → C2 → C3）
2. 空の状態から各差分を順に適用する

スナップショットは保存しない。全ての状態はコミットチェーンの走査で計算される。

## サブモジュール（Submodule）

動画編集ソフトにおける個々の編集要素を表す。クリップ、エフェクト、トランジション、字幕、トラック、グローバルエフェクトの 6 種類がある。

### サブモジュールの独立性

各サブモジュールは独立したバージョン履歴を持つ。

- 自分だけのコミットチェーンを持つ
- 自分だけのブランチを持つ（あるサブモジュールは `experiment` ブランチ、別のサブモジュールは `main` ブランチにいる、という状態が正常）
- マージやリベースもサブモジュール単位で行われる

この設計は動画編集の実態に即している。エフェクトのパラメータ調整を試したいとき、クリップの配置まで巻き込んでブランチを切る必要はない。

### 依存関係

サブモジュール間には依存関係を定義できる。例えば、あるエフェクトは特定のクリップに依存する。この依存情報はグローバルタイムラインでの整合性チェックに使われる。

---

## ブランチ（Branch）

コミットチェーン上の特定の地点を指す名前付きポインタ。

### ブランチの実体

ブランチはコミット ID への参照に過ぎない。ブランチが「存在する」とは、BranchStore 内にそのエントリがあるということ。ファイルシステム上にブランチごとのディレクトリが作られるわけではない。

### main ブランチ

サブモジュール作成時に自動的に作られる既定のブランチ。削除できない。

### ブランチの分岐

新しいブランチを作ると、分岐元コミットの地点に新しいポインタが置かれる。この時点では両ブランチは同じコミットを指している。その後どちらかにコミットを追加すると、ブランチが分岐する。

```
          ┌── [B1] ──→ [B2]  (feature)
[C1] ──→ [C2]
          └── [C3] ──→ [C4]  (main)
```

### ブランチ切り替え

ブランチを切り替えると、サブモジュールの HEAD がそのブランチの先端コミットに変わる。Calix はメタ情報のみを管理するため、ファイルシステム上の変更は一切発生しない。HEAD ポインタが移動するだけである。

---

## グローバルタイムライン

サブモジュールをまたいだ操作の順序を記録する仕組み。

### 目的

サブモジュールは独立しているが、動画全体としての編集順序は意味を持つ。「クリップを配置してからエフェクトを追加した」という順序関係をグローバルタイムラインが保証する。

### 構造

タイムラインは連番のシーケンス番号を持つエントリの列。各エントリは以下を記録する。

- どのサブモジュールで
- どのコミットが
- どの種別のイベントとして（通常コミット / マージ / リベース）
- いつ発生したか

### イベント種別

- **Commit** — 通常のコミット
- **Merge** — マージコミット。第二親の情報を含む
- **Rebase** — リベース完了。書き換え前後のコミット ID 対応表を含む

### 依存関係チェック

マージやリベースの際に、依存先サブモジュールの最新コミットが自サブモジュールの最新コミットよりタイムライン上で後ろにある場合、警告を返す。これはエラーではなく警告であり、操作は続行できる。
